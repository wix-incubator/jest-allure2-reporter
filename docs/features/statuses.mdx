# Statuses

:::caution

This website version refers to the unreleased version of `jest-allure2-reporter` and is not yet available
anywhere. Please use GitHub docs for the latest stable version.

:::

Jest Allure 2 Reporter provides a clear indication of your test cases' statuses, allowing you to immediately see if they have _passed_, _failed_, _broken_, been _skipped_, or are of _unknown_ status.

## Handling broken tests

If you are using assertions (matchers, expectations) that throw errors directly
instead of using Jest's `expect` API, and you want to treat these errors as broken tests,
there is a way to do it.

In your Jest config file, add an options object to the `jest-allure` reporter config with
`errorsAsFailedAssertions` set to `true`:

```diff
module.exports = {
  // ...
  reporters: [
    // ...
-   'jest-allure',
+   ['jest-allure', {
+     errorsAsFailedAssertions: true,
+   }],
  ],
};
```
