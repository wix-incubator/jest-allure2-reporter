JestAllure2ReporterOptions

JestAllure2ReporterOptions: { errorsAsFailedAssertions: boolean; getEnvironmentInfo: boolean | (() => Promise<Record<string, string>>); overwriteResultsDir: boolean; packageName: string; resultsDir: string }
  • Defined in JestAllure2ReporterOptions.ts:19

Options to use with jest-allure2-reporter package in Jest config

example

{ // ... reporters: [ 'default', [ 'jest-allure2-reporter', { resultsDir: 'allure-results', packageName: 'my-package', errorsAsFailedAssertions: true, }, }

Type declaration

  • errorsAsFailedAssertions: boolean

    Treat thrown errors as failed assertions. By default, the reporter distinguishes between failed assertions and thrown errors. The former are reported as FAILED tests, the latter as BROKEN tests.

    default

    false

  • getEnvironmentInfo: boolean | (() => Promise<Record<string, string>>)

    Getter function to extract environment information from the test environment. By default, the environment information is extracted from the process.env object. Use boolean false to disable environment information.

    default

    true

  • overwriteResultsDir: boolean

    Whether the reporter should delete the results directory before running tests.

    default

    true

  • packageName: string

    Add an extra label to each test case with the package name. Helpful when running tests from multiple packages in a monorepo.

    default

    import('package.json').name

  • resultsDir: string

    Path to the directory where the report will be generated.

    default

    /allure-results
